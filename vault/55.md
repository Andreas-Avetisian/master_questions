# Asymmetric vs. symmetric cryptography: For what problems would you use which? Why? What are the advantages/disadvantages? What algorithms do you know?

Symmetric and asymmetric cryptography solve different problems.

## Symmetric Cryptography

Symmetric cryptography uses the **same secret key** for encryption and decryption.

- Best use: encrypting **large amounts of data quickly**, for example files, disks, network traffic, audio, or video streams.
- Advantage: very fast.
- Advantage: keys are usually relatively short.
- Disadvantage: the secret key must be exchanged securely in advance.
- Disadvantage: in group communication the number of required shared keys grows quickly.
- Disadvantage: it is **not suitable for digital signatures**.

Algorithms named in the cited slides include:

- **AES**
- **Triple DES / 3DES**

## Asymmetric Cryptography

Asymmetric cryptography uses a **public key** and a **private key**. One key is distributed publicly, while the private key stays secret.

- Best use: **key exchange**, **digital signatures**, certificates, and communication where parties do not already share a secret.
- Advantage: easy key distribution.
- Advantage: fewer key pairs are needed.
- Advantage: suitable for digital signatures.
- Disadvantage: much slower than symmetric cryptography.
- Disadvantage: longer keys are usually required.
- Disadvantage: current methods in the slides are **not quantum-computer secure**.

Algorithms named in the cited slides include:

- **RSA**
- **ElGamal / DLIES**
- **ECIES**
- **DSA**
- **ECDSA**
- **DH / Diffie-Hellman**
- **ECDH**

## When To Use Which

- Use **symmetric cryptography** for the actual protection of data, because it is efficient.
- Use **asymmetric cryptography** when you need to distribute keys safely or create and verify digital signatures.

In practice, both are usually combined: asymmetric cryptography establishes or protects a session key, and symmetric cryptography encrypts the real data.
