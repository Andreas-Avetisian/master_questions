# Explain the concept of l-diversity, provide two different approaches and discuss drawbacks and practical implementation problems. What is the difference to t-closeness?

**l-diversity** is an extension of **k-anonymity**. Its goal is that sensitive values inside each equivalence class should be distributed well enough that an attacker still cannot learn too much about a person even after narrowing the person down to that class.

The key question is therefore: what counts as **well distributed**?

## Approaches

- **Distinct l-diversity:** each equivalence class must contain at least `l` different sensitive values.
- **Entropy l-diversity:** not only the number of values matters, but also how evenly they are distributed; entropy is used as the measure.
- **Recursive (c,l)-diversity:** the most frequent sensitive value must not dominate too strongly compared with the less frequent ones.

Two common approaches to mention in an exam are **distinct l-diversity** and **entropy l-diversity**.

## Drawbacks And Practical Problems

- Distinct l-diversity can still leak sensitive information if one value is much more likely than the others.
- Different sensitive values may still have almost the same meaning, for example different types of cancer.
- Data quality often drops strongly because more generalization or suppression is needed.
- It becomes even harder when there is more than one sensitive attribute.
- Rare values are a major problem because they make l-diversity difficult or impossible to achieve.
- Entropy l-diversity is mathematically more complex and hard to interpret in practice.
- Even belonging to an equivalence class can already reveal something sensitive, for example that a person is a cancer patient.

## Difference To T-Closeness

l-diversity only looks at the distribution **inside one equivalence class**. It does not check whether that class distribution is similar to the distribution in the **whole table**.

This is exactly why the **similarity attack** is possible: a class may satisfy l-diversity, but the values can still all be very similar, for example all indicating low income.

**t-closeness** fixes this by requiring that the distribution of the sensitive attribute in each equivalence class must stay close to the distribution in the whole table, with a maximum distance of `t`.

So:

- **l-diversity:** enough diversity inside the class
- **t-closeness:** class distribution must also stay close to the global distribution

t-closeness is therefore stronger, but also even harder to achieve in practice.
