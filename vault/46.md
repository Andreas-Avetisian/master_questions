# Define k-anonymity and explain record linkage and homogeneity attack. What can be done against the latter?

## K-anonymity (slides 104-105, 114)

The lecture motivates k-anonymity by noting that many attributes are *quasi-identifiers* (QIs): they are not direct identifiers like a name or SSN, yet combined they can still single out a person. The task is to keep such QIs useful for analysis while preventing re-identification.

**Basic idea (slide 105)**: *"Change the quasi-identifiers so that at least **k** data sets become indistinguishable from each other with respect to quasi-identifiers. Of course, (direct) identifiers must be removed."*

In practice this is achieved by:

- **Generalization**: reducing the granularity of attributes (e.g. replace birthday with birth year, replace a ZIP code with its first three digits).
- More complex generalization strategies.
- Other approaches like **outlier deletion** and **unlinking attributes**.

The course places k-anonymity under "data-based anonymization, making data indistinguishable" (slide 104), as opposed to evaluation-based approaches like differential privacy.

A subtle refinement (slide 114) is sometimes called **(x,y)-anonymity**: *"at least k records from different people have to be in each equivalence class."* This avoids the trivial case where the same person contributes all k tuples.

## Record linkage (slide 100)

Record linkage is the **simple inference attack** that motivates k-anonymity in the first place. The definition on the slide:

- *"Attacker has external information for inference."*
- *"First defined by Sweeney in 2001."*
- *"~75% of US citizens can be identified by combining Sex, ZIP code, Birthdate."*

In other words, even without direct identifiers, an attacker who holds an auxiliary dataset (e.g. a voter roll) can **link records** from the "anonymized" dataset to real individuals by joining on the shared quasi-identifiers. The classic Sweeney result shows that just three QIs are enough to re-identify most of the US population, which is why k-anonymity insists that every QI-tuple appears in at least *k* records.

## Homogeneity attack (slides 115-117)

Once k-anonymity is in place, the attacker can no longer pinpoint *which* record belongs to a given person. But the lecture flags the next problem on slide 116:

*"We might not be able to discern the exact record belonging to a user, but we gather (sensitive) knowledge on them. Also called **attribute linkage**."*

The mechanism is simple: if an equivalence class of k records is k-anonymous with respect to the QIs but all k records share the *same* value for a sensitive attribute, the attacker learns that sensitive value for the target with 100% certainty, without ever identifying the exact row.

The canonical example on slide 117: *"Based on the inference attack that served as the motivator for k-anonymity, an attacker can establish 100% that Daniela Marple has lung cancer"*, because every record in Daniela's equivalence class carries the diagnosis "lung cancer".

So the homogeneity attack exploits the **lack of diversity in the sensitive attribute** inside an equivalence class. K-anonymity protects the *identity*, but not the *attribute*.

## Countermeasures against the homogeneity attack

The lecture offers two layered fixes, both sitting on top of k-anonymity.

### L-diversity (slides 119, 120, 126)

**Basic idea (slide 119)**: *"The values of sensitive data are well distributed with respect to each equivalence class generated by the k-anonymity criterion."* The crux is how to operationalise "well distributed"; the lecture discusses three variants.

- **Distinct l-diversity (slide 120)**: each equivalence class must contain at least *l* different values for the sensitive attribute. Also known as *p-sensitive k-anonymity*.
- **Entropy l-diversity**: the distribution of sensitive values inside each class must be not only diverse but also *sufficiently even* (measured by entropy). Strongest version, but very strict and often impractical.
- **Recursive (c,l)-diversity (slide 126)**: *"the value that occurs most frequently in the sensitive attributes must not occur too frequently"*, formally $f_1 < c \cdot \sum_{i=l}^{m} f_i$. A pragmatic compromise between distinct and entropy l-diversity.

### T-closeness (slides 131, 132)

L-diversity has a remaining weakness (slide 131): *"l-diversity measures only the distribution within the equivalence classes. No statement whether this distribution matches the distribution in the overall table."* A class can be "diverse" yet still strongly skewed compared to the population (e.g. all "low income" values).

**T-closeness (slide 132)** closes this gap: *"An equivalence class satisfies the t-closeness criterion if the distance between the distribution of a sensitive attribute in the equivalence class does not differ from the distribution in the whole table by more than a factor t."* In other words, each class must look **statistically similar to the overall population** on the sensitive attribute, which neutralises homogeneity-style inferences even when the class is technically diverse.

## Short summary

**K-anonymity** is a data-based anonymization criterion: change the quasi-identifiers so that each record is indistinguishable from at least k-1 others with respect to those QIs, after removing direct identifiers. **Record linkage** is the motivating attack (Sweeney 2001): an attacker joins the released data with external sources on QIs and re-identifies individuals, demonstrably ~75% of US citizens from just sex, ZIP and birthdate. The **homogeneity attack** (attribute linkage) shows that k-anonymity alone is not enough: if all k records in an equivalence class share the same sensitive value, the attacker learns that value for the target without identifying the row. Defences in the lecture are **l-diversity** (distinct, entropy, recursive c,l), which forces sensitive-attribute diversity inside each class, and **t-closeness**, which additionally requires the class distribution to stay close to the overall population distribution.
