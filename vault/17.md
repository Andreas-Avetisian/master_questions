# Describe the most important encoding mechanisms for categorical data.

Categorical variables usually have to be transformed into numerical form, because most machine learning algorithms cannot process raw categories directly.

| Encoding | Basic idea | Main point for exams |
| --- | --- | --- |
| **Label encoding** | Assign each category an integer value (`0 ... n`). | Very simple, but for nominal data it introduces an artificial order that usually has no real meaning. |
| **Ordinal encoding** | Assign integers according to a real rank or order. | Suitable for ordered categories such as `low < medium < high`. |
| **One-hot encoding** | Create one binary column per category; `1` means the category is present, `0` otherwise. | Best-known method for nominal data because it avoids fake order, but it increases the number of columns. |
| **Binary encoding** | First map categories to integers, then represent those integers as binary digits across several columns. | Needs fewer columns than one-hot, so it is useful when a feature has many different categories. |
| **Mean (target) encoding** | Replace each category with the mean target value of that category. | Compact encoding, especially for high-cardinality features, but it depends on the target variable. |

Important distinctions:

- **Label encoding** is easy, but problematic for unordered categories like hair color, because the numbers suggest mathematical distances or order that do not actually exist.
- **Ordinal encoding** is the correct variant when the categories really do have a meaningful sequence.
- **One-hot encoding** is often preferred for unordered categories. Its drawback is dimensionality: many categories create many new features. In practice, one dummy column is often omitted, leaving `n-1` columns.

Exam takeaway:

- Use **ordinal encoding** for ordered categories.
- Use **one-hot encoding** for unordered nominal categories.
- Be careful with **plain label encoding** for nominal data.
- **Binary** and **mean encoding** are useful alternatives when the number of categories is large.
